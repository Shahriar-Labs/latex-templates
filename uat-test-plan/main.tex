\section*{\color{primaryNavy}1. Introduction \& Objectives}
\sectionline

This User Acceptance Testing (UAT) Plan defines the strategy, scope, and execution framework for validating NimbusPay v2.4 against business requirements. The objective is to ensure the release meets stakeholder expectations and is fit for production deployment.

\vspace{6pt}

\begin{tcolorbox}[
  enhanced,
  colback=bgLight,
  colframe=accentBlue,
  boxrule=0.8pt,
  left=12pt,
  right=12pt,
  top=8pt,
  bottom=8pt,
  sharp corners,
  before skip=6pt,
  after skip=6pt
]
  \textbf{Primary Objectives:}
  \begin{itemize}[nosep, leftmargin=*]
    \item Validate that all functional requirements defined in BRD-NIM-2026-03 are implemented correctly.
    \item Confirm end-to-end business workflows execute without failure under realistic conditions.
    \item Verify that non-functional requirements (performance, security, usability) meet acceptance thresholds.
    \item Obtain formal stakeholder sign-off before production release.
  \end{itemize}
\end{tcolorbox}

\section*{\color{primaryNavy}2. Scope}
\sectionline

\subsection*{2.1 In Scope}

\begin{itemize}[nosep, leftmargin=*]
  \item User authentication module with role-based access control (RBAC).
  \item Payment processing engine --- credit card and ACH transactions.
  \item Transaction reporting dashboard with export capabilities.
  \item Integration with external fraud detection service (Vertex Shield API).
  \item Administrative user management and audit logging.
\end{itemize}

\subsection*{2.2 Out of Scope}

\begin{itemize}[nosep, leftmargin=*]
  \item Mobile application (iOS/Android) --- tested separately under NIM-UAT-MOB-2026.
  \item Hardware security module (HSM) certification --- pending external audit.
  \item Legacy batch processing module --- deprecated from v2.4.
\end{itemize}

\section*{\color{primaryNavy}4. Test Scenarios \& Acceptance Criteria}
\sectionline

\begin{tcolorbox}[
  enhanced,
  colback=white,

  sharp corners,
  title={\textbf{TC-001: User Login \& Multi-Factor Authentication}},
  fonttitle=\bfseries\color{white},
  colbacktitle=accentBlue
]
  \textbf{Priority:} Critical \hfill \textbf{Assigned To:} Sarah Lin, UAT Tester\\[4pt]
  \textbf{Precondition:} Valid user credentials exist in the staging identity provider.\\[4pt]

  \begin{tcolorbox}[
    enhanced,
    colback=bgLight,
    colframe=borderGray,
    boxrule=0.5pt,
    left=10pt,
    right=10pt,
    top=6pt,
    bottom=6pt,
    sharp corners,
    title={Test Steps},
    fonttitle=\bfseries\color{primaryNavy}
  ]
    \begin{enumerate}[nosep, leftmargin=*]
      \item Navigate to \texttt{https://uat.nimbuspay.internal/login}.
      \item Enter valid email and password; submit the login form.
      \item Enter the 6-digit OTP received via authenticator app.
      \item Verify redirection to the dashboard.
      \item Attempt login with an expired OTP (30-second window exceeded).
      \item Attempt login with invalid credentials three times consecutively.
    \end{enumerate}
  \end{tcolorbox}

  \vspace{4pt}

  \begin{tcolorbox}[
    enhanced,
    colback=bgLight,
    colframe=statusPass,
    boxrule=0.5pt,
    left=10pt,
    right=10pt,
    top=6pt,
    bottom=6pt,
    sharp corners,
    title={\passmark\ Acceptance Criteria},
    fonttitle=\bfseries\color{statusPass}
  ]
    \begin{itemize}[nosep, leftmargin=*]
      \item Successful login redirects to dashboard within 2\,seconds.
      \item Expired OTP is rejected with message ``Authentication code expired. Request a new code.''
      \item After three failed attempts, the account is locked for 15\,minutes and an alert is logged.
      \item All login events (success and failure) are recorded in the audit trail.
    \end{itemize}
  \end{tcolorbox}
\end{tcolorbox}

    fonttitle=\bfseries\color{statusPass}

    \begin{itemize}[nosep, leftmargin=*]
      \item Payment authorization completes within 3\,seconds for transactions under \$500.
      \item Fraud detection response is received and logged; high-risk transactions are flagged for manual review.
      \item Declined transactions display a clear reason code (e.g., ``Insufficient funds,'' ``Do not honor'').
      \item Partial refund is processed within one business-day settlement window and reflected in the ledger.
      \item All payment events are recorded with a unique transaction reference ID.
    \end{itemize}
  \end{tcolorbox}
