% --- SECTION 1: EXECUTIVE SUMMARY ---
\section{Executive Summary}
On July 14, 2026, at 08:15 UTC, the Vertex core database platform experienced a critical resource exhaustion failure, resulting in 45 minutes of complete API service degradation and timeout issues for end users. The incident was triggered by a rapid, compounding memory leak in the primary database server's buffer cache. The root cause was traced to a newly deployed analytics query that executed a full-table scan without appropriate database indexing. The incident was resolved at 09:00 UTC after database promotion of a read-only replica and temporary isolation of the offending query.

\subsection{Key Metrics}
\begin{itemize}[label=\textcolor{SecondaryColor}{\tiny\faCircle}, leftmargin=15pt, nosep]
  \item \textbf{Total Downtime:} 45 minutes (08:15 UTC -- 09:00 UTC)
  \item \textbf{Impacted Users:} Approximately 12,000 active customer sessions on Vertex Cloud
  \item \textbf{Financial SLA Impact:} Estimated \$35,000 USD in SLA credit obligations
  \item \textbf{Recovery Time Objective (RTO) Status:} Exceeded target of 15 minutes by 30 minutes
\end{itemize}

% --- SECTION 3: PROBLEM INVESTIGATION ---
\section{Problem Investigation}
Post-incident analysis revealed that the crash occurred due to memory starvation (Out-Of-Memory) on the primary database host. The profiling data collected during the outage showed that the active memory pool was consumed entirely by query execution plans associated with the \texttt{/v2/\allowbreak analytics/\allowbreak transactions} endpoint. 

A newly deployed backend service update included a reporting query that attempted to aggregate transaction histories by customer ID and creation timestamp. However, because the database migration script did not apply a composite index across the \texttt{customer\_id} and \texttt{created\_at} columns, PostgreSQL was forced to execute a sequential scan of 45 million rows. This query, executed concurrently by hundreds of active sessions, led to severe thrashing and eventual system shutdown.

% --- SECTION 6: LESSONS LEARNED ---
\section{Lessons Learned}
This incident highlights the risk of scaling database operations without formal governance and automated enforcement. Key lessons include:
\begin{enumerate}[label=\textbf{\arabic*.}, leftmargin=15pt, nosep]
  \item Relying on manual checklists for schema changes is insufficient at our current scale.
  \item Query profiling must be integrated into pre-production environments to catch full-table scans.
  \item High-severity alerts should trigger automated failover systems faster than the manual intervention timeframe.
\end{enumerate}

% --- SECTION 4: 5-WHYS ANALYSIS ---
\section{Root Cause Analysis (5-Whys)}
To determine the systemic root cause of the database failure, the engineering team performed a structured 5-Whys analysis:

\begin{tcolorbox}[colback=LightNeutral, colframe=PrimaryColor, arc=4pt, title=\textbf{5-Whys Analysis}, fonttitle=\large\bfseries, boxrule=1.5pt]
  \textbf{Problem Statement:} The production database host crashed under load, resulting in 45 minutes of complete service disruption.
  
  \vspace{0.2cm}
  \begin{itemize}[label=\textcolor{AccentColor}{\small\faArrowRight}, leftmargin=15pt, itemsep=4pt]
    \item \textbf{Why 1? } Why did the database server crash? \\
      \textit{Answer: } System memory was exhausted, triggering the Linux Kernel's Out-Of-Memory (OOM) killer.
    \item \textbf{Why 2? } Why was the system memory exhausted? \\
      \textit{Answer: } A single query scanning the large \texttt{transactions} table grew exponentially in memory under concurrent requests.
    \item \textbf{Why 3? } Why did the query require so much memory? \\
      \textit{Answer: } The query lacked a composite index, forcing PostgreSQL to perform sequential full-table scans.
    \item \textbf{Why 4? } Why was there no index on these columns? \\
      \textit{Answer: } The backend feature was deployed without a database migration review by a Database Reliability Engineer.
    \item \textbf{Why 5? } Why was the database migration review bypassed? \\
      \textit{Answer: } Our CI/CD deployment pipeline did not enforce automated checks for database migrations, and the development sprint checklist lacked a mandatory sign-off block for schema changes. \textbf{[Root Cause]}
  \end{itemize}
\end{tcolorbox}
